Test Driven Developpement

  • 1. Ecrire un test
  • Test first: Qu'est ce que je dois faire ?
  • 2. Lancer les tests et vérifier que le test est KO
  • On vérifie son harnais de sécurité
  • 3. Ecrire le code le plus simple qui fait passer les tests
  • Ce n'est pas grave si c'est du code "sale"
  • 4. Lancer les tests
  • 5. Refactoring & Nettoyage
  • duplication, renommage, déplacement, patterns, etc.
  • Répéter

Bénéfices

  • Test automatisés pour garantir la non régression
  • Test first: meilleure testabilité
  • Documentation exécutable
  • Moins de temps perdu à débugger
  • Design: on ne fait que ce qui est nécessaire (pragmatisme)
  • Agréable : flow, feedback loop rapide, confiance pour la non régression
